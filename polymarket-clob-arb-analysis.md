## 1. 第一性原理：Box Arbitrage 的数学基础

### 1.1 策略定义

```
操作：
  ① 在二元预测市场上，同时买入 UP 和 DOWN 两种 token
  ② 持有至结算
  ③ 胜出方每股份赔付 $1.00，败方价值归零

数学约束（忽略 Gas）：
  Profit = $1.00 − Price(UP) − Price(DOWN)
  Profit > 0 ⟺ Price(UP) + Price(DOWN) < $1.00
```

**这是唯一的盈亏条件。** 无论市场如何变化、无论 BTC 涨跌，只要买入成本之和 < $1.00，策略就必然盈利。

### 1.2 为什么这是"无风险套利"

| 情形 | UP 价格 | DOWN 价格 | 总成本 | 赔付 | 利润 |
|:-----|:-------:|:---------:|:------:|:----:|:----:|
| 🟢 理想套利 | $0.45 | $0.45 | $0.90 | $1.00 | **+$0.10** |
| 🟡 微利 | $0.51 | $0.49 | $1.00 | $1.00 | $0.00 |
| 🔴 亏损 | $0.99 | $0.99 | $1.98 | $1.00 | **−$0.98** |
| 🔴 亏损 | $0.60 | $0.55 | $1.15 | $1.00 | −$0.15 |

**关键洞察：利润仅取决于买入成本之和与 $1.00 的差值，与市场方向、波动率、任何外部因素无关。这是一个纯粹的数学套利。**

### 1.3 理论上的可行性

```
假设前提：
① 存在二元结果的市场（UP / DOWN，互斥且完备）
② 每个 token 可以在独立的订单簿上交易
③ 结算机制可靠（胜者赔 $1.00）

在这些前提下，策略是理论上 100% 可行的。
数学上不存在任何使策略"不可能"的内在障碍。
```

### 2.2 观测到的关键模式

```
                  UP Token              DOWN Token
              ┌─────────────┐        ┌─────────────┐
              │  asks        │        │  asks        │
              │  0.99 (8k)   │        │  0.99 (8k)   │
              │  0.98 (1.6k) │        │  0.98 (1.6k) │
              │  0.97 (10k)  │        │  0.97 (10k)  │
              │  ...         │        │  ...         │
              │  0.01 (8k)   │        │  0.01 (8k)   │
              ├─────────────┤        ├─────────────┤
              │  bids        │        │  bids        │
              │  0.01 (8k)   │        │  0.01 (8k)   │
              │  0.02 (1.6k) │        │  0.02 (1.6k) │
              │  0.03 (10k)  │        │  0.03 (10k)  │
              │  ...         │        │  ...         │
              │  0.99 (8k)   │        │  0.99 (8k)   │
              └─────────────┘        └─────────────┘

模式特征：
① 完全对称 — UP 和 DOWN 的 ask/bid 结构几乎完全相同
② 双向深度 — 做市商同时在 buyside 和 sellside 提供全深度报价
③ 恒定 spread — bid ≈ 0.01, ask ≈ 0.99（spread ≈ 0.98）
④ 大额深度 — 每个价位都有数千股份的流动性
```

### 2.3 多市场验证

| 市场 | UP ask | UP bid | DOWN ask | DOWN bid | 模式 |
|:-----|:------:|:------:|:--------:|:--------:|:----:|
| BTC 15min | 0.99 | 0.01 | 0.99 | 0.01 | 对称 |
| BTC 1M GTA VI | 0.999 | 0.001 | 0.999 | 0.001 | 对称 |
| 其他 crypto 二元 | 0.99 | 0.01 | 0.99 | 0.01 | 对称 |

**结论：跨市场一致。这不是个别现象，而是 CLOB 做市商的系统性策略。**

### 2.4 做市商的视角：他们才是真正的 Box Arbitrager

```
做市商同时在两边的订单簿上报价：
┌─────────────────────────────────────────────────────┐
│                                                       │
│  UP Token        ─3─ 做市商 ─2─  DOWN Token         │
│                                                       │
│  ① 在 UP 挂 bid=0.01 (有人卖 → MM 买入 @ $0.01)     │
│  ② 在 DOWN 挂 bid=0.01 (有人卖 → MM 买入 @ $0.01)  │
│  ③ 合计成本: $0.02                                   │
│  ④ 到期赔付: $1.00                                    │
│  ⑤ 做市商利润: $0.98 / 份                            │
│                                                       │
└─────────────────────────────────────────────────────┘

做市商同时在 UP 和 DOWN 的 ask 侧挂单 0.99:
  若有人在 UP 买 (MM 卖 @ $0.99) +
  若有人在 DOWN 买 (MM 卖 @ $0.99)
  = MM 收 $1.98，支付 $1.00，获利 $0.98

做市商在两边同时赚取 spread — 无论方向，都是稳赚。
```

## 3. 结论是否成立？

### 3.1 "策略不可行"结论的精确表述

| 断言 | 成立？ | 说明 |
|:-----|:------:|:-----|
| "作为 taker 买入两边不可行" | ✅ **成立** | 最佳 ask 之和 = 1.98，远超 1.00 |
| "策略理论上不可能" | ❌ **不成立** | 数学上完全可能，问题在操作层面 |
| "Box Arbitrage 在任何市场上均不可能" | ❌ **不成立** | 在 AMM 模式、或不同做市商竞争下可能 |
| "无法在 Polymarket 套利" | ❌ **不成立** | 不能用 taker 方式，但 maker 方式可行 |

### 3.2 精准结论

> **在 Polymarket CLOB 上，使用"同时以市价买入 UP 和 DOWN token"的方式，Box Arbitrage 不可行。**
>
> **但策略本身并非不可能。将操作方向从"taker（吃 ask）"反转为"maker（吃 bid）"，则转为可行——只不过变为被动做市，竞争激烈。**

### 3.3 策略失败的路标

```
ASK 侧（我们的方向）：     BID 侧（做市商的方向）：
┌──────────────┐            ┌──────────────┐
│ 吃 ask = 花费 │            │ 吃 bid = 收入 │
│ 0.99 + 0.99  │            │ 0.01 + 0.01  │
│ = $1.98      │            │ = $0.02      │
│ 赔付 $1.00   │            │ 赔付 $1.00   │
│ 净亏 −$0.98  │            │ 净赚 +$0.98  │
│              │            │              │
│ ❌ 我们在做  │            │ ✅ 应该做的  │
└──────────────┘            └──────────────┘
```

## 4. 策略调整方案

### 4.1 方案 A：Bid 侧 Box Arbitrage（Maker 模式）

**思路：** 不主动买入，而是挂限价 bid 等待对手方卖出。

```
操作（每个 15 分钟周期）：
  ① 挂 UP bid @ $0.45，数量 5 股
  ② 挂 DOWN bid @ $0.45，数量 5 股
  ③ 如果双方均成交：成本 $0.90 × 5 = $4.50，赔付 $5.00，利润 $0.50
  ④ 如果仅一方成交：产生方向性敞口，需对冲或接受风险
```

**优势：**
- 理论上利润空间巨大（$0.98 spread 中分一点）
- 机械执行，无需预测方向

**劣势：**
- 成交概率极低（做市商 bid=0.01 下单量极大，零售卖单会优先流向 MM）
- 如果出价高于做市商 bid（如 0.02 vs 0.01），可能成为唯一的有效 bid 并获得成交，但成本翻倍
- 单边成交风险（挂了 UP bid 被吃了，DOWN 没被吃 → 手上有裸露的多头仓位）

**可行性评估：** 5/10 — 逻辑正确，但成交效率极低，零售卖单流量稀少。

### 4.2 方案 B：单边限价单 + 即时对冲

**思路：** 只在有一方被吃时，立刻在另一方挂限价 ask 对冲。

```
示例流程：
  ① 监控订单簿，当 UP 的 bid 被大量吃掉时（说明有卖方压力）
  ② 如果 UP bid 从 0.01 下移到 0.005 或消失
  ③ 立刻买入 UP（以 $0.01 或更低价限价买入）
  ④ 同时在 DOWN 挂 bid=0.99（但不会成交，因为 DOWN ask 也是 0.99）
```

这个等价于变成了方向性交易，脱离了无风险套利框架。

**可行性评估：** 3/10 — 已偏离 Box Arbitrage 核心，变成了方向性投机。

### 4.3 方案 C：新市场窗口套利

**思路：** BTC 15 分钟市场每 900 秒开一次。在第一个做市商订单到达之前的极短窗口内，订单簿可能是空的或只有少数零售挂单。

```
操作：
  ① 计算下一个市场的时间戳 T = floor(now/900) * 900 + 900
  ② 在 T 毫秒时，同时发送两笔市价买单
  ③ 如果两边都没有做市商挡在 0.99，可能有低价成交的机会
```

**实际测试数据：** 从收集到的几个市场来看，新市场一开启做市商就已经挂好了 0.99 ask。这可能是 Polymarket 的部署管道在 market 状态变为 "open" 之前已经准备好了订单。

**可行性评估：** 3/10 — 理论上有窗口，但实际中做市商的部署管道极快（毫秒级 vs. 我们的 HTTP API 延迟），个人 bot 几乎不可能抢到。

### 4.4 方案 D：事件驱动的 Spread 波动套利

**思路：** 做市商的 spread 并非始终恒定。在极端事件（大量交易涌入、链上拥堵、做市商重新部署）时，spread 可能短暂偏离正常值。

但数据表明，BTC 15 分钟市场流动性极好，spread 偏离几乎不发生。

**可行性评估：** 2/10 — 需要极高频的订单簿快照，且偏离罕见的微观事件。

## 5. 替代套利策略

### 5.1 策略矩阵

| # | 策略 | 类型 | 可行性 | 复杂度 | 预期年化 | 
|:-:|:-----|:-----|:------:|:------:|:--------:|
| 1 | **跨平台套利** | 无风险 | **8/10** | 中 | 中 |
| 2 | **双向限价做市** | 做市 | **7/10** | 高 | 高 |
| 3 | **AMM 残余套利** | 统计 | 4/10 | 高 | 低 |
| 4 | **结算前确定性套利** | 事件 | **6/10** | 低 | 中 |
| 5 | **统计方向交易** | 统计 | 5/10 | 高 | 变化 |
| 6 | **相对价值套利** | 统计 | 5/10 | 中 | 中 |

### 5.2 策略详解

#### 🔵 策略 1：跨平台套利（推荐优先级最高）

**原理：** 相同的二元预测结果在不同平台上可能存在定价差异。

```
候选平台：
  ├── Polymarket CLOB (当前)
  ├── Jupiter Forecast (Solana, 调研中)
  ├── dYdX Prediction Markets
  └── Kalshi (如 API 开放)

套利条件：
  在平台 A 买入 UP/YES 的成本 + 在平台 B 买入 DOWN/NO 的成本 < $1.00

为什么可行：
  - 不同平台有独立的做市商，流动性不共享
  - 做市商只需关注自己的订单簿，不需要跨平台维持一致定价
  - 即使两边 spread 都很大，加起来可能仍然 < $1.00
```

**示例：**
```
Polymarket CLOB:  UP ask=0.99, DOWN ask=0.99 → 没法做
但:
Jupiter Forecast:  YES ask=0.40 → 买
Polymarket CLOB:   NO ask=0.50 → 买
总成本: $0.90 < $1.00 → 每股利润 $0.10 ✓
```

**实现复杂度：**
- 需要 Jupiter 和 Polymarket 两个 SDK
- 跨平台仲裁（哪个平台买哪一方）的决策逻辑
- 不需要高频或实时，普通 REST API 足够

**可行性评估： 8/10** — 前提是 Jupiter 上有对应的 BTC 方向性市场，且定价模式不同。

#### 🔵 策略 2：双向限价做市 Grid Strategy

**原理：** 不在极端价位挂单，而是在 spread 中间地带同时挂双向限价单。

```
UP token:
  ├── BID 0.30 挂 5 股 (比 MM 的 0.01 高很多!)
  └── ASK 0.70 挂 5 股 (比 MM 的 0.99 低很多!)

DOWN token:
  ├── BID 0.30 挂 5 股
  └── ASK 0.70 挂 5 股

结果：
  - 若双向 BID 均成交: 成本 $0.60, 赔付 $1.00 → 利润 $0.40
  - 若双向 ASK 均成交: 到期支付 $0.00 (因为是卖方), 收 $1.40 → 利润 $0.40
  - 若 UP 的 bid 和 DOWN 的 ask 均成交: 做多了一个 UP、做空了一个 DOWN → delta 中性(?)

等待时间：5-10 分钟到几小时
```

**优势：**
- 价格明显优于做市商报价（bid 比 0.01 好 30 倍！）
- 零售卖方可能会优先选择你的报价 -> 更容易成交
- 不对市场方向有任何观点 bias

**劣势：**
- 成交率仍然无法保证
- 需要管理部分成交带来的持仓敞口
- 面临 adverse selection 风险（为什么有人愿意以 0.30 卖给你？可能因为信息不对称）

**可行性评估： 7/10** — 需要精确的仓位跟踪和风险管理。

#### 🔵 策略 3：AMM 残余流动性套利

**历史背景：** Polymarket 曾经使用 AMM (FPPMM) 作为主要交易机制。2024 年迁移到 CLOB 后，是否仍有遗留的 AMM 流动性池或仅部分市场保留了 AMM？

**实际状态：** 当前 BTC 15 分钟市场是纯 CLOB，未发现 AMM 残余流动性。这个策略在 2024 年之前是可行的，但现在可能已经过时。

**可行性评估： 4/10** — 仅适用于特定旧市场，新市场不适配。

#### 🔵 策略 4：结算前确定性交易

**原理：** BTC 15 分钟市场每 900 秒结算一次。在市场即将结算的最后一分钟，如果 BTC 价格已经大幅偏离开盘价，结果已经确定性偏高。

```
示例：
  BTC 从 $85,000 涨到 $85,500 → UP 几乎确定获胜
  上次报价: UP bid=0.01 (已过时，因为结果已确定)
  此时 UP 应该价值 $1.00，但做市商可能还没来得及更新报价
```

但这个策略的问题在于：做市商有实时数据（他们也需要这个数据来管理风险）。他们的 bid/ask 会被算法自动调整到贴近实际预期概率。在市场很"确定"的时候，获胜方的价格会快速逼近 $1.00。

**可行性评估： 6/10** — 需要极低的延迟（交易所在最后一分钟可能已经停止订单）。

#### 🔵 策略 5：统计方向交易

**原理：** 构建 BTC 短期价格预测模型，与 Polymarket 的概率定价进行对比，找出定价错误。

这不属于无风险套利，而是统计套利。需要承担方向风险。

**可行性评估： 5/10** — 需要复杂的 ML 模型和大量回测数据。

#### 🔵 策略 6：相对价值套利

**原理：** 找出一组相关市场，计算它们的定价是否满足跨市场约束。

```
示例（需要多个互斥结果的市场）：
  MMA 第 1 轮 KO 胜 vs 第 2 轮 KO 胜 vs ... 
  所有这些互斥结果的总价格应该 ≤ $1.00
  如果 > $1.00，存在套利机会
```

但 BTC 15 分钟是典型的简单二元市场，无法应用。

**可行性评估： 5/10** — 仅适用于复杂多元市场。

## 附录 A：数学证明补充

### A.1 Box Arbitrage 是无风险套利的严格证明

```
假设：
  - 二元市场，两个互斥完备的结果 A 和 B
  - Token A 在结果 A 时偿付 $1，否则 $0
  - Token B 在结果 B 时偿付 $1，否则 $0
  - p_A = 买入 Token A 的单价
  - p_B = 买入 Token B 的单价
  - n = 各买入的股份数

收获矩阵：
              结果 A 发生    结果 B 发生
  Token A       $n            $0
  Token B       $0            $n
  合计          $n            $n        ← 恒为 $n

总成本 = n × p_A + n × p_B

条件 Profit > 0:
  $n > n × (p_A + p_B)
  1 > p_A + p_B                  ← 仅取决于 sum of prices
  
QED. 不论二元结果的概率分布如何，不论时间推移，利润仅取决于
买入成本之和与 $1.00 的差值。这是数学恒等式。
```

### A.2 CLOB 为何阻止了这个恒等式的利用

```
AMM 模式下:
  p_A + p_B 恒等于 1 (因为 AMM 曲线 x*y=k 的约束)
  → 理论上总可以找到稍 < 1.00 的买点（含slippage）
  → Box Arbitrage 可行

CLOB 模式下:
  p_A 和 p_B 由做市商独立决定
  p_A = ask_A (最佳卖价)
  p_B = ask_B (最佳卖价)
  → 做市商设置 ask_A ≈ 1.00 - bid_A，但 ask_A + ask_B 无约束
  → 当 ask_A = 0.99 + ask_B = 0.99 = 1.98 时，无法盈利
  → Box Arbitrage 不可行 (作为 taker)

但反向操作:
  p_A = bid_A (最佳买价) = 0.01
  p_B = bid_B (最佳买价) = 0.01
  → bid_A + bid_B = 0.02 < 1.00
  → 作为 maker 可以盈利
  → 但需要等待对手方主动卖出，无法主动买入
```
